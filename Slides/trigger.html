Triggers in FAAS

Modelling Triggers in Serverless Application

Trigger

  • Aplication is a list of the services. app(app1, [service1, service2]).
  • Each service is a set of the stateless functions. func(sum, [x,y], 1, rust, 10). func(div, [z,z], 2, python, 20).
  • Each service has an event that should trigger activation of the functions. service(service1, triggerX, sum, 1, [ubuntu], [eu]). service(service2, triggerY, div, 1, [sql], [eu]).

Modelling Trigger

  • We are using triggers to activate functions in a service when some event occurs. trigger(contactsService, triggerZ, notification).
  • A rule is part of a trigger and we use it for an association from trigger to action.
  • Rule contains triggerID and its configuration parameters. rule(triggerZ, [eu, us]).

Fire Trigger

  • When the invocation of action completes, the invocation fires the associated completion trigger.
  • when the completion trigger of `a` fires, it schedules an invocation of action `b`.
