Covert channels

  • Mechanisms for signaling information through a computing system are known as channels.
  • Channels that exploit a mechanism whose primary purpose is not information transfer are called covert channels; they pose the greatest challenge in preventing improper information leaks.

Implicit Flows – Leak 1

  • if(hi > 0) { lo = 99 }
  • There is a leak here, because the type system allow usage of lower secret label under high guard.
    • “Leaks the sign of guard, not its value"

Implicit flows - Solution 1

  • ctx(AOp, ife(FId, P1, P2), L) :-
    func(FId, Args, _, _, _),
    labelF(AOp, Args, L),
    ctx(AOp, P1, L),
    ctx(AOp, P2, L).
  • The problem solved during the implementation in class by enforcing "guard", "then" and "else" parts having the same label that is highest among all of them.

Implicit Flows – Leak 2

  • if(hi > 0) {
    skip;
    skip;
    } else { skip; }
  • The problem is execution time, the attacker can get sign of hi because then and else part have different execution time.
    • “Leaks the sign of hi, not its value"

Implicit flows - Solution 2

  • ctx(AOp, ife(FId, P1, P2), TUnits, L) :- func(FId, Args, _, _, _),
    labelF(AOp, Args, L),
    ctx(AOp, P1,TUnitsT, L),
    ctx(AOp, P2,TUnitsE, L),
    TUnitsT == TUnitsE, TUnits is TUnitsT.
  • Solution is checking time units of then and else part but under then and else there could be another If then else, while loop, try catch, or sequential programs.
  • We need to change other indicator to be able to get sequential sum of programs

Implicit flows - Solution 2 - Functions

  • ctx(AOp, FId, TUnits,L) :-
    func(FId, Args, _, _, TUnits), % TUnits = Computational time for FId
    labelF(AOp, Args, L).
  • Just get time unit of function and assign it as time unit of program

Implicit flows - Solution 2 - Sequential Program

  • ctx(AOp, seq(P1, P2), SeqTUnits,L) :-
    ctx(AOp, P1, FstTUnits, L),
    ctx(AOp, P2, SndTUnits, L),
    SeqTUnits is FstTUnits + SndTUnits.
  • Sum up time units of first and second program and assign it as sequential time units

Implicit flows - Solution 2 - Try Catch

  • ctx(AOp, trc(P1, P2), TUnitsTRC ,L) :- ctx(AOp, P1, TTUnits, L),
    ctx(AOp, P2, CTUnits, L),
    TTUnits == CTUnits,
    TUnitsTRC is TTUnits.
  • We assumed try and catch have same logic with the if then else, therefore, we checked time units of try and catch and assigned time units of try as time units of try catch

Implicit Flows – Solution 2 – While

  • ctx(AOp, whl(FId, P), TUnitsW, L) :-
    func(FId, Args, _, _, CTUnits),
    labelF(AOp, Args, L),
    guardCheck(L),
    ctx(AOp, P, BTunits, L),
    TUnitsW is CTUnits + BTunits.
  • Time units of while must represent in program as infinity.

Implicit Flows – Leak 2

  • while(ts) {
    skip;
    skip;
    }
  • The problem comes from termination and non-termination.
  • Type system allows while to have secret labeled arguments inside the guard.
    • “Leaks the sign of ts, not its value"

Termination Channels - Solution

  • ctx(AOp, whl(FId, P), TUnitsW, L) :-
    func(FId, Args, _, _, CTUnits),
    labelF(AOp, Args, L),
    guardCheck(L),
    ctx(AOp, P, BTunits, L),
    TUnitsW is CTUnits + BTunits.
  • We considered to add a guard checker to check label inside the guard, therefore, if there is a forbidden label it will fail.

Termination Channels - Solution

  • guardCheck(X):- not(X = ts),
    not(X = s),
    not(X = ts_eu),
    not(X = ts_us),
    not(X = s_eu),
    not(X = s_us).
  • The forbidden labels defined by the operator under guardCheck predicate
